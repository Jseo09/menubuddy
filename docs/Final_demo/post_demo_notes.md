After the April 29 demo, the MenuBuddy system successfully demonstrated a complete end-to-end RAG pipeline, including menu ingestion, retrieval, grounded answer generation with citations, as well as both pre-generation and post-generation safety mechanisms. The system performed particularly well in handling irrelevant questions, such as “Who is the CEO?”, by correctly issuing early refusals before retrieval. It also provided strong explainability through citation-supported answers.

During the demo, however, some valid menu-related questions were flagged due to the retrieval module pulling data from multiple restaurant menus, which introduced noise into the context and reduced grounding accuracy. Additionally, source labeling in the interface was not fully refined, leading to less clear and less polished outputs for users.

For future improvements, we would limit retrieval to a single menu source per session to ensure cleaner and more relevant context. We would also improve source labeling for better readability and user experience, and further tune the validator to reduce false negatives on valid answers.
