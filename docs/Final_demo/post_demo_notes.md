Post-April 29th demo, the MenuBuddy system demonstrated a robust RAG pipeline from scratch to finish, including menu ingestion, menu retrieval, grounded answer generation with citations, pre-generation safety measures, and post-generation measures. In particular, the MenuBuddy system excelled at answering irrelevant questions (for example, “Who is the CEO?”) with an early refusal and was quite explainable through answer generation that provided citations for support. Nevertheless, during the demo, some valid menu-related questions were marked up since the retrieval module retrieved information about multiple restaurants’ menus, causing noise in context that negatively impacted grounding certainty. Moreover, the source labeling in the interface was not fully cleaned, resulting in unappealing outputs for end users. As possible improvements, we would limit retrieval to one menu source per session and improve our source labeling practices to provide more accurate labels. We would also fine-tune the validator to lower false negatives on valid answers.
